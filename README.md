# Hasnane Nawaz Portfolio

Premium neo-brutalist portfolio for Hasnane Nawaz, built with Next.js 15, TypeScript, Tailwind CSS, Framer Motion, Lenis, GSAP, Lucide Icons, and shadcn-style UI primitives.

## Architecture

```text
/app
  /api/contact/route.ts      Contact validation endpoint
  globals.css                Theme variables, base styling, brutal utilities
  layout.tsx                 Metadata, JSON-LD, providers, shared shell
  page.tsx                   Homepage composition
  robots.ts                  SEO robots route
  sitemap.ts                 SEO sitemap route
/components
  /layout                    Navbar and footer
  /providers                 Theme, scroll, loading, cursor wrappers
  /sections                  Hero, About, Experience, Skills, Projects, Achievements, Testimonials, Contact
  /shared                    Cursor, loading screen, smooth scroll
  /ui                        shadcn-style button, reveal, section heading, magnetic button
/hooks
  use-mounted.ts             Client mount helper
/lib
  portfolio-data.ts          CV-derived content model
  site-config.ts             SEO, profile, contact, JSON-LD
  utils.ts                   Classname helper
  zod-lite.ts                Tiny validation utility for contact route
/public
  site.webmanifest           App metadata
/styles
  README.md                  Style system notes
```

## Component Strategy

The homepage is assembled from focused section components so each section can be reordered, tested, or replaced independently. CV facts live in `lib/portfolio-data.ts`, which keeps content separate from presentation and makes future updates simple.

Reusable UI primitives are intentionally small:

- `Button` follows shadcn conventions with variants and `asChild`.
- `Reveal` handles intersection-observer entrance animations.
- `MagneticButton` adds premium pointer interaction without affecting accessibility.
- `SectionHeading` keeps editorial rhythm consistent across the page.

## UI/UX System

The visual system uses neo-brutalist principles: hard borders, assertive shadows, high-contrast blocks, expressive display typography, visible grid texture, and intentionally tactile interaction states. The tone is professional but memorable: marketer-as-operator, not template portfolio.

Design tokens:

- Background: warm paper `#FFF8E8`
- Ink: `#111111`
- Electric accent: `#B8FF2C`
- Coral accent: `#FF6B57`
- Cyan accent: `#5CE1E6`
- Radius: tight, mostly `6px`
- Shadow: offset black brutal shadows
- Type: Inter for body, Space Grotesk for display

## Homepage Sections

1. Hero: name, role, positioning statement, CTA buttons, core metrics.
2. About: storytelling cards connecting law, research, content, and growth.
3. Experience: interactive-feeling timeline cards built from CV roles.
4. Skills: categorized neo-brutalist skill blocks.
5. Projects: case-study cards based on measurable CV outcomes.
6. Achievements: stats and professional proof points.
7. Testimonials: placeholder structure ready for verified quotes.
8. Contact: validated form, email handoff, phone, location.

## Animation Strategy

- Lenis controls smooth scrolling.
- Framer Motion handles section reveal, magnetic buttons, and hero entrance.
- GSAP powers the loading screen transition.
- Motion is scoped and respectful of `prefers-reduced-motion`.
- Hover states use transform and shadow changes for tactile brutalist feedback.

## SEO, GEO, and AI Search

- Metadata API configured in `app/layout.tsx`.
- JSON-LD Person schema includes role, contact, location, and areas of expertise.
- Sitemap and robots routes are generated by Next.js.
- Content repeats important entity terms naturally: Hasnane Nawaz, social media marketer, growth strategist, community growth, lead generation, Durg, Chhattisgarh.
- Internal anchor links make the one-page site easy for recruiters and crawlers to scan.

## Performance Targets

The build avoids heavy imagery and keeps animation libraries isolated to client components. Fonts use `next/font`, sections use semantic HTML, and the contact form works progressively with a server route plus mail client handoff.

Recommended checks before deployment:

```bash
npm install
npm run typecheck
npm run build
npm run dev
```

Deploy on Vercel by importing the repository and setting the production URL in `lib/site-config.ts`.
